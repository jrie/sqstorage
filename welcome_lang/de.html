Willkommen

bei sqStorage oder kurz sqS.

Dies ist sehr empfehlenswert, da Sie eine grundlegende Einführung erhalten und auch weitere Informationen, wie Sie helfen können, sqS noch großartiger zu machen oder es einfach an Ihre Bedürfnisse anzupassen.

Grundlegende Einführung

Dieser Text "Grundlegende Einführung" richtet sich an Erstbenutzer von sqS. Für den Fall, dass Sie neu sind, gibt es eine kurze Anleitung, was Sie tun können.

Kurze Zusammenfassung

In sqS können Sie auf einfache Weise Speicher (Bereiche, Räume, Orte, Truhen, Schränke, Kisten...), Inventare, Sammlungen aller Art anlegen. Zum Beispiel Ihre Film-/Spiel-/Musik-/Anwendungssammlung oder alles andere, was Sie gerne im Auge behalten möchten. Es ist ganz einfach: Sie nennen es, sqStorage speichert es!

Grundlagen

Darüber hinaus können Sie Artikel jeglicher Art mit grundlegenden Informationen wie Titel/Label, Menge, Kommentare, Seriennummer und Artikelnummern als Standard-/Basisfelder hinzufügen. Außerdem haben Sie die Möglichkeit, Artikelkategorien und Unterkategorien zu verwenden, um die Artikel weiter einzugrenzen.

Zum Hinzufügen von Lagern legen Sie einen Artikel an und geben den gewünschten Lagernamen ein. Das Lager wird dann erstellt und der Artikel wird in diesem Lager gespeichert. Wenn Sie einen weiteren Artikel hinzufügen, können Sie den neuen Lagerort auswählen oder einen weiteren Lagerort erstellen.

Erweiterbare und benutzerdefinierte, dynamische, Datenfelder

sqS ist aber nicht auf diese Informationen beschränkt und kann von Ihnen selbst erweitert werden.

Wenn Sie z.B. "Spiele-Schlüssel" oder allgemein "Seriennummern/Lizenzen" benötigen, können Sie einfach ein benutzerdefiniertes Datenfeld hinzufügen, das diese Informationen für die Kategorie "Spiele" und auch "Anwendungen" speichert. Oder Sie benötigen einen Kaufpreis, den Sie ebenfalls als benutzerdefiniertes Feld hinzufügen können. Das Beste daran ist, dass Sie zuordnen können, auf welche Art oder Kategorie von Artikeln sich das benutzerdefinierte Datenfeld bezieht. So werden diese Informationen nicht für Artikelkategorien angezeigt, bei denen Sie nicht möchten, dass die Daten eingegeben oder angezeigt werden.

Benutzersystem

Wenn Sie Ihre sqS-Installation vor fremden Zugriffen schützen wollen, können Sie dies durch die Aktivierung eines Benutzersystems tun. Sie können Administratoren, Benutzer und Gäste hinzufügen, die jeweils eigene Berechtigungen haben.

Grundsätzlich können Administratoren alles tun, z.B. Benutzer anlegen. Benutzer dürfen Artikel und Kategorien oder benutzerdefinierte Felder hinzufügen, erstellen und löschen. Gäste können nur sehen, was gespeichert ist, sie haben keine weiteren Rechte.

Rückmeldungen

Feedback ist einfach das, was dieses Projekt antreibt! Wirklich, es ist wichtig.

Sie müssen kein Programmierer oder Software-Ingenieur oder ähnliches sein, um Feedback zu geben. Jeder, der bereit ist, 5 Minuten seiner Zeit zu opfern, um ein Problem in der Software (Bug) zu melden, einen Anwendungsfall zu erklären oder einfach nur mitzuteilen, dass Sie die Software erfolgreich nutzen, ist mehr als willkommen.

Sie können Vorschläge, Empfehlungen und allgemeines Feedback per e-Mail an Jan senden. Er wird gerne darauf antworten.

Aber Sie können auch Fehler oder Funktionswünsche online auf der Github-Seite des Projekts melden: https://github.com/jrie/sqstorage/issues.

Im weiteren

Nachdem sie die arkanischen Mysterien dieser Software verinnerlicht haben, fangen Sie an in dem Sie Ihren ersten Gegenstand, in Ihrem ersten Lagerort, mit ihrer ersten Kategorie anlegen. Und einen zweiten und ein dritten. Sie werden schnell den Dreh raus haben.

Und falls Sie einmal nicht weiterkommen, lesen Sie unter "Rückmeldungen" nach und schreiben eine e-Mail oder einen Fehlerbericht, wenn eine benötigte Funktion fehlt.

Das Team hinter sqStorage hilft Ihnen gerne dabei etwas zu schaffen.
